## Bugs Found and Fixed During This Run

### BUG: T2 — Disk validation fails when QEMU holds qcow2 lock

**File:** `scripts/phone-shortcuts/tasks/10-UOM-Start`
**Root cause:** `qemu-img info` cannot read a qcow2 image that is locked by a running QEMU process. Returns error, script treats it as "corrupted disk" and exits.
**Fix:** Skip disk validation when `uom_qemu_running` returns true. Only validate disk when QEMU is stopped.
**Status:** Fixed, deployed, tested — T2 PASS after fix.

### BUG: T8 — Guest poweroff requires privilege escalation (doas/sudo not configured)

**File:** `scripts/phone-shortcuts/90-UOM-Stop` and `bin/uom-qemu-phone` (launcher `cmd_stop`)
**Root cause:** Guest Alpine has `doas` installed but `/etc/doas.conf` is root-owned with no `permit nopass` for wheel group. `PermitRootLogin no` in sshd_config. Neither `doas poweroff` nor `sudo poweroff` works from `uom` user.
**Fix (widget):** Added QEMU monitor `system_powerdown` (ACPI) fallback when SSH poweroff fails.
**Fix (launcher):** Added QEMU monitor `system_powerdown` fallback + `quit` after 10s if process still alive. Reduced timeout from 60s to 30s.
**Status:** Fixed, deployed, tested — T8 PASS after fix. ACPI shutdown completes in ~2s.

### BUG: T8 — /dev/tty absent in non-interactive mode

**File:** `scripts/phone-shortcuts/90-UOM-Stop:31`
**Root cause:** `read -r _confirm </dev/tty 2>/dev/null` fails when stdin is piped (no TTY). Fallback `read -r _confirm` works correctly from pipe. Non-fatal — the script still reads the confirmation.
**Status:** Known limitation. Non-interactive usage (pipework, widget) works via fallback. Interactive TTY mode works on device.

### BUG: T9 — Widget SSH timeout too short for cold boot

**File:** `scripts/phone-shortcuts/tasks/10-UOM-Start:18`
**Root cause:** `SSH_READY_TIMEOUT=120` may not be enough for TCG-emulated cold boot (can take 3-5 minutes depending on device load).
**Recommendation:** Increase to 300s for cold boot, or detect first-boot vs warm-start.
**Status:** Not yet fixed — test eventually passed after external wait.

## Minor Observations (Non-blocking)

1. **Guest tmux empty after cold boot** — Expected. No tmux server starts by default in the guest.
2. **`qemu-img info` shows "(locked by QEMU or unavailable)" in status** — Cosmetic; expected when VM is running.
3. **`ps -A | grep qemu` returns nothing on Android** — Android's `ps` format differs. Launcher uses `/proc/PID/cmdline` scan instead (works correctly).
4. **`whoami` returns `localhost` on phone** — Android `hostname` command behavior. Not a bug.

Both fixes are deployed to phone and in the repo. The system is stable for continued development.
