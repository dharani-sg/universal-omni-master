# M11 Roadmap — Atomic Rollback + Boot-to-Snapshot

**Status:** Next (post M10-A / **v0.10.0**)  
**Tag target:** `v0.11.0`  
**Depends on:** M3 (`omni-boot`), M7 (deploy layout), M10-A (`omni-snapshot`)

---

## Why M11, not M10-B

| Option | Verdict |
| ------ | ------- |
| **M10-B** | Reject for this feature. M10-A already released as **v0.10.0** with a green gate (syntax + 20 snapshot tests + M1–M9 regressions). A “B” phase implies residual lifecycle polish, not a new restore/boot product surface. |
| **M11** | **Chosen.** Boot-to-snapshot + atomic restore is a *new capability class*: it couples snapshot inventory with bootloader entry generation and root-subvol surgery. That is a major milestone, not a patch on M10-A. |
| **M12** | Fish TUI (original M10 alternate track), deferred until restore is CLI-complete. |

Install-time `src/deploy/rollback.sh` remains **partition/subvol cleanup during failed deploy**. M11 restore is **runtime** system recovery from `@auto_*` snapshots — different code path, different risk model.

---

## Problem statement

M10-A can **create**, **list**, **prune**, and **hook** snapshots. It cannot yet:

1. Present a snapshot as a **bootable menu entry** (rootflags / subvolid).
2. **Atomically replace** the live `@` (or default subvolume) with a chosen snapshot.
3. Offer **one-shot** “boot this snapshot once, then return to default”.
4. Guarantee a **pre-restore safety snapshot** so a bad restore is recoverable.

M11 closes that loop: *snapshot → boot entry → boot once → permanent rollback*.

## Architecture

```
omni-snapshot (CLI)
├── create/list/prune/sweep/periodic     # M10-A (exists)
├── boot-entry  [add|list|remove|sync]   # M11-A
├── boot-once   <snap>                   # M11-C
└── rollback    <snap> [--mode swap|inplace]  # M11-B

omni-boot (backend only)
├── boot_write_snap_entry()              # extend GRUB + systemd-boot
├── boot_remove_snap_entry()
└── boot_set_oneshot_default()           # EFI LoaderEntryOneShot / GRUB next

src/snapshot/
├── common.sh / prune.sh / periodic.sh   # M10-A
├── boot_entry.sh                        # M11-A NEW
└── restore.sh                           # M11-B NEW

src/deploy/
└── snapshot_install.sh                  # M11-D: enable boot-entry sync on deploy
```

**Hard rules (carry forward from M10-A):**

- Mutation guard: exit **126** under `OMNI_SYSROOT`.
- Non-Btrfs: graceful skip (return 0 + warning), never abort package hooks.
- Free-space gate before any new snapshot (including pre-restore safety snap).
- POSIX `/bin/sh` only; no bashisms.

## Phases

### M11-A — Bootloader entry generation

**Goal:** For any managed snapshot name, generate a bootloader entry that boots with that subvolume as root.

| Item | Detail |
| ---- | ------ |
| systemd-boot | BLS Type #1 under `loader/entries/omni-snap-<name>.conf` with `options root=UUID=… rootflags=subvol=<path>,ro` (or `subvolid=`) |
| GRUB | Drop-in `/etc/grub.d/42_omni_snapshots` **or** managed snippet regenerated by `omni-snapshot boot-entry sync`; `menuentry --id omni-snap-…` |
| Discovery | Resolve snapshot → Btrfs ID + path via `_snap_btrfs_root()` + `btrfs subvolume list -o` (reuse R2 fix) |
| Kernel/initrd | Reuse **current** `/boot` kernels (snapshot is root FS only unless separate `/boot` layout documented) |
| CLI | `omni-snapshot boot-entry add <name>`, `list`, `remove <name>`, `sync` (reconcile entries ↔ existing snaps; drop orphans on prune) |
| Prune hook | After `snap_prune`, remove boot entries whose snaps no longer exist |

**Acceptance**

- [ ] Fixture with fake ESP + GRUB cfg: entry written with expected title/options
- [ ] Mutation guard on all write paths
- [ ] `sync` removes stale `omni-snap-*` entries after prune
- [ ] Non-Btrfs / missing bootloader → skip 0

### M11-B — Atomic live restore

**Goal:** Permanently roll the running system’s default root subvolume back to a chosen RO snapshot.

| Mode | Mechanism | When |
| ---- | --------- | ---- |
| **swap** (default) | 1) create pre-restore RO snap of current `@`  2) create RW snap of target under temp name  3) rename/set-default in a tight sequence  4) fstab/default-subvol update  5) reboot required | Prefer when layout uses named `@` at top-level |
| **inplace** | `btrfs property set -ts … ro false` on a *copy* of the snap, then `set-default` | Fallback when rename is unsafe |

**Safety sequence (mandatory)**

1. `_snap_guard_mutation` + `_snap_require_btrfs` + free-space check  
2. Verify target exists and is RO (`@auto_*`)  
3. Create `pre-restore` safety snap (category `emergency` or `manual`)  
4. Optionally `boot-entry add` for the safety snap  
5. Perform swap / set-default  
6. Emit audit event (`omni-audit` / healer-compatible log line)  
7. Print reboot instruction; **do not** auto-reboot unless `--reboot`  

**Explicit non-goals**

- Restoring while `/` is the only subvol with no `@` layout (document + refuse with clear error)  
- Rolling back `/boot` or ESP contents (out of band; document risk if kernels diverge)  
- Overloading `deploy/rollback.sh` (keep install-time recovery separate)

**CLI**

```text
omni-snapshot rollback <snap-name> [--mode swap|inplace] [--no-safety-snap] [--reboot]
omni-snapshot rollback --list-candidates
```

**Acceptance**

- [ ] Dry-run / fixture path never mutates host  
- [ ] Safety snap created before any set-default  
- [ ] Failure mid-sequence leaves previous default intact **or** documents recoverable state  
- [ ] Integration with M8: emergency orphans still swept by `sweep`

### M11-C — Boot-once (one-shot entry)

**Goal:** Next boot uses snapshot X; subsequent boots return to previous default without permanent rollback.

| Backend | Mechanism |
| ------- | --------- |
| systemd-boot | EFI `LoaderEntryOneShot` (efivarfs; reuse M3 `_efi_*` patterns) + ensure entry exists (M11-A) |
| GRUB | `grub-reboot <id>` / `saved_entry` one-shot |

**CLI:** `omni-snapshot boot-once <snap-name>`

**Acceptance**

- [ ] Second boot returns to prior default (fixture simulation of EFI var / grubenv)  
- [ ] Missing entry auto-created via M11-A before one-shot set  

### M11-D — Deploy + gate

| Deliverable | Notes |

## Risk register (audit before impl)

| ID | Risk | Mitigation |
| -- | ---- | ---------- |
| R1 | Kernel in `/boot` newer than modules in old snap → unbootable | Document; optional “boot-once first” recommendation; later: snap `/boot` if on Btrfs |
| R2 | `subvol=` path wrong relative to FS root (M10 R2 class) | Reuse `_snap_btrfs_root()`; store both path and subvolid in entry metadata comment |
| R3 | set-default without reboot leaves mixed mounts | Always require reboot for permanent rollback; print fstab check |
| R4 | Prune deletes snap still referenced by default/oneshot | Refuse prune of default subvol; clear oneshot if target pruned |
| R5 | GRUB drop-in broken by distro grub-mkconfig | Prefer 42_omni_snapshots executable + `boot-entry sync` after mkconfig |
| R6 | LUKS root | Pass through existing cryptdevice= from live/default entry; never invent UUID |
| R7 | Dual-boot foreign OS entries clobbered | Only touch `omni-snap-*` / `--id omni-snap-*` namespace |
| R8 | AC-only / degraded SATA host ENOSPC mid-restore | Free-space gate + pre-restore snap; abort if either fails |

## Suggested implementation order

1. Interface stubs + mutation guards + tests that fail closed  
2. M11-A systemd-boot path (cleaner unit tests)  
3. M11-A GRUB path  
4. M11-A `sync` + prune integration  
5. M11-B safety snap + set-default (fixture-only first)  
6. M11-C oneshot  
7. M11-D deploy install + full gate
